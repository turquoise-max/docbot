# [2026-05-20] 능동적 페어 라이팅 도입 및 문서 에디터 UX/동기화 이슈 해결

## 1. 이슈 배경 및 문제점

문서 작성 및 수정 과정에서 다음과 같은 세 가지 주요 문제가 식별되었습니다.

1. **AI의 수동적 태도 및 더미 데이터 생성 (Hallucination)**
   - 템플릿을 생성할 때 AI가 사용자의 의도를 묻지 않고 임의의 가상 데이터(더미 데이터)로 빈칸을 채워버리는 현상 발생.
   - 기존에는 사용자가 직접 부족한 부분을 지적하고 질문해야만 AI가 반응하는 수동적인 구조.
2. **사용자 편집 데이터 유실 및 부적절한 UI 문구**
   - AI가 `writeDocument` 도구를 호출해 문서를 업데이트할 때, 사용자가 에디터에서 직접 수정한 텍스트가 무시되거나 AI 임의대로 요약/삭제되는 현상.
   - 문서를 단순히 '수정'하는 단계임에도 채팅 패널 UI에는 항상 "문서 초안 작성 완료"라고 고정되어 출력됨.
3. **에디터 렌더링 깜빡임(Flicker) 및 스크롤 튕김 현상**
   - AI가 문서를 재생성하여 에디터에 적용할 때 화면이 하얗게 비워졌다가 다시 채워지며 심하게 깜빡이는 현상.
   - 문서가 로드되면서 사용자가 보던 뷰포트(스크롤) 위치가 무조건 최상단으로 초기화되는 심각한 UX 저하 발생.

## 2. 해결 과정 및 구현 상세

### 2.1. 능동적 페어 라이팅(Proactive Pair Writing) 루프 구축
에디터의 복잡한 DOM을 직접 건드리는 대신, AI의 프롬프트를 전면 개편하여 똑똑하게 핑퐁 대화를 유도하도록 수정했습니다.

- **`src/components/chat/components/ChatMessageItem.tsx` 수정:**
  - `HIDDEN_ANALYZE_PROMPT`를 수정하여 기존의 "요약 리포트" 대신, "문서에서 비어있거나 보완이 필요한 섹션을 하나 찾아 사용자에게 먼저 질문하라"고 지시.
- **`src/lib/ai/prompts.ts` (System Prompt) 개편:**
  - `editorContext`를 바탕으로 **3단계 행동 우선순위** 설정:
    1. `[회사명]` 등 플레이스홀더 발견 시 질문
    2. 부실한 더미 데이터 섹션 발견 시 구체화 질문 제안
    3. 일반적인 추상적 수정 요청 대응
  - 이를 통해 AI가 스스로 문서를 스캔하고 부족한 부분을 먼저 채우자고 제안하는 진정한 의미의 '페어 라이팅' 경험 완성.

### 2.2. 사용자 편집 내용 100% 보존 및 동적 UI 문구 적용
`editorContext`가 실시간으로 에디터의 변경 사항을 잘 추적하고 있다는 사실을 확인한 뒤, AI가 이 문맥을 무시하지 못하도록 규칙을 강화했습니다.

- **절대 보존 프롬프트 추가 (`prompts.ts`):**
  - "사용자가 작성해둔 기존 내용은 토씨 하나 틀리지 않고 100% 그대로 보존하라. 특정 섹션만 수정/추가하고 나머지는 그대로 유지하라"는 강력한 규칙을 최상단에 배치.
- **동적 완료 문구 (`src/lib/ai/tools/index.ts`):**
  - `writeDocument` 도구의 파라미터 스키마에 `summary`를 추가하고 설명(description)을 변경.
  - 신규 초안일 경우 "문서 초안 작성 완료", 기존 문서 수정일 경우 "문서 수정 완료"로 상황에 맞게 UI에 텍스트가 출력되도록 유도.

### 2.3. 무중단 덮어쓰기 및 스크롤 위치 강제 복원 (핵심 UX 개선)
가장 까다로웠던 에디터 화면 깜빡임과 스크롤 튕김 현상을 Syncfusion API의 특성을 우회하여 해결했습니다.

- **`open()` 대신 `selectAll()` + `paste()` 적용 (`src/components/editor/SyncfusionDocEditor.tsx`):**
  - `loadDocument` 호출 시 기존의 `editor.open()`은 렌더러 전체를 초기화하므로 화면이 깜빡입니다.
  - 문서가 비어있을 때는 `open()`을 쓰되, 이미 내용이 있는 문서의 갱신일 경우 전체 선택 후 붙여넣기(`editor.selection.selectAll()` -> `editor.editor.paste()`) 방식을 사용하여 렌더링 초기화 없이 텍스트만 자연스럽게 교체하도록 수정했습니다.
- **DOM 스크롤 강제 복원 로직 구현:**
  - 덮어쓰기 직전에 뷰포트 컨테이너(`container_viewerContainer`)의 `scrollTop` 값을 변수에 저장.
  - `paste()` 완료 직후 100ms 대기(setTimeout)를 통해 렌더링 사이클을 넘긴 뒤, 뷰포트의 `scrollTop`을 저장된 값으로 원복.
  - 커서는 `moveToDocumentStart()`로 깔끔하게 리셋하여 사용자가 문서를 편집/확인하던 뷰를 완벽하게 유지.
- **CSS 트랜지션 추가 (`src/app/globals.css`):**
  - `.e-documenteditor-container` 에 `transition: opacity 0.3s ease-in-out;` 을 추가해 시각적인 부드러움을 더함.

## 3. 결과 및 기대 효과
- **완벽한 대화형 작성 흐름:** AI가 일방적으로 더미 데이터를 던지지 않고, 사용자와 티키타카를 하며 문서를 완성해 나갑니다.
- **사용자 신뢰도 향상:** 직접 수정한 공들인 문장들이 AI의 재생성 과정에서 날아가는 현상이 근절되었습니다.
- **압도적인 UX 개선:** 문서 전체가 다시 쓰여지더라도 화면 번쩍임이나 스크롤 위치 손실이 없어(마치 VS Code에서 파일이 핫리로딩 되는 것처럼), 실시간으로 문서가 매끄럽게 수정되는 경험을 제공합니다.
